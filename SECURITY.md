# Security policy — fastmvc-db-models

## Reporting a vulnerability

**Do not** open public GitHub issues for security reports.

Email: **sengarsinghshivansh@gmail.com**

Include: description, impact, steps to reproduce, affected versions, and suggested fix if any.

## Supported versions

Security fixes are applied to the latest minor release line; upgrade to the newest patch release.

## General practices

- Keep dependencies updated.
- Do not commit secrets; use environment variables or a secrets manager.
